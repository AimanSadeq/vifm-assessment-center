# Self-Report Scales — Clusters 2–8 (Likert + Forced-Choice/TIRT)

**Companion to:** `competency-self-report-cluster1.md` (the worked model, with
full scale/scoring/TIRT method and implementation notes — **not repeated here**).

For each competency: a **normative Likert** set (4 items, one reverse-keyed) and
a **forced-choice** statement pool (3 statements) arranged into balanced blocks
per cluster.

**Likert scale:** 1 Strongly disagree … 5 Strongly agree. **(R)** = reverse-score
(6 − raw). Competency score = mean of 4 items.

**Forced-choice:** each block shows statements from *different* competencies,
matched for desirability; respondent marks MOST / LEAST like them; scored via
**Thurstonian IRT** (Brown & Maydeu-Olivares, 2011). Codes (A–E) map to the
competencies listed in each cluster. *Block tables show codes; at delivery,
statement text is shown in randomised order.*

> Production steps (empirical desirability matching, mixed keying, calibration N,
> fit checks) are in the Cluster 1 doc and apply identically here.
