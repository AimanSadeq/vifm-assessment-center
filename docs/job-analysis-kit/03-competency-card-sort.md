# 03 — Competency Card-Sort

**Purpose:** Capture, per role, (a) how **critical** each draft competency is and
(b) the **structural edits** that resize the framework. This instrument is the
primary mechanism for letting the evidence change the set.

**Duration:** 15–20 min · **Materials:** 38 competency "cards" (name + 1-line
definition from `docs/competency-framework-v2-design.md` §4). Print, or use a
spreadsheet/whiteboard.

---

## Part A — Criticality sort

Ask the SME to place every competency card into one band **for this role**:

| Band | Meaning |
|---|---|
| **Essential** | Role fails without it |
| **Important** | Strongly contributes to success |
| **Useful** | Helpful but not differentiating |
| **Not relevant** | Not required for this role |

Recording sheet:

| Competency | Essential | Important | Useful | Not relevant |
|---|:---:|:---:|:---:|:---:|
| Forward Strategy Setting | ☐ | ☐ | ☐ | ☐ |
| Commercial & Market Awareness | ☐ | ☐ | ☐ | ☐ |
| *(…all 38…)* | | | | |

> **Optional forced distribution (Q-sort):** to prevent everything landing in
> "Essential," cap the Essential band (e.g., max 8–10 cards). Recommended once
> you have enough SMEs.

## Part B — Structural edits *(do not skip)*

After sorting, ask directly:

1. **Merge** — "Are any of these really the *same thing* for this role? Which
   should be combined?" → list pairs.
2. **Split** — "Is any single card actually *two different things* that you'd
   rate separately?" → list.
3. **Add** — "Is anything this role *critically* needs **missing** from the
   deck?" → name it + a one-line definition. *(Watch for GCC-banking-specific
   items: regulatory/Sharia awareness, AML/financial-crime vigilance, client
   confidentiality, conduct/customer-fairness.)*
4. **Drop** — "Which cards simply don't belong for this role?"
5. **Rename** — "Does any card's *name* not match how you'd describe it here?"

Recording:

| Edit type | Competencies involved | SME rationale |
|---|---|---|
| Merge | | |
| Split | | |
| Add | | |
| Drop | | |
| Rename | | |

---

## Aggregation (after all SMEs)

- **Criticality:** convert bands to scores (Essential 3 / Important 2 / Useful 1
  / Not relevant 0); take the mean per competency per role.
- **Structural edits:** tally how many SMEs proposed each merge/split/add/drop.
  An edit endorsed by a **majority** of SMEs is actioned per the README §5
  decision rules; minority suggestions are logged for review.
- Record every actioned change in the **analysis log** (the originality/validity
  audit trail).
